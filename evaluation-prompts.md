# 网红管理平台编码评测 Prompt 集（第二套）

> 项目：Influencer Management Platform（React 18 + FastAPI + MySQL 8 + Docker）  
> 共 47 题：Feature迭代 ×20 · 0-1构建 ×20 · 代码测试 ×3 · 代码重构 ×2 · 工程化 ×2  
> 说明：本套题目与 `coding-tasks-prompts.md` 无功能重叠，均可通过前端页面验证（测试/重构/工程化题除外）。

## Feature迭代（20）

### F-01
分类管理页「相关网红数」列始终显示为 0，且前端用该字段判断是否允许删除，导致有网红关联的分类仍可被误删。请修复字段映射与删除禁用逻辑，并在该列数字大于 0 时以主题色高亮、等于 0 时灰色展示；同时为每个分类行增加「查看网红」文字链接，点击后跳转网红列表并自动带上对应 `category_id` 筛选且分页重置为第一页。用 operator 登录后确认美妆等种子分类的网红数与数据库一致，点击链接后列表仅展示该分类网红，有网红的分类删除按钮应为禁用状态。

### F-02
网红列表页目前仅通过下拉框选择平台，切换效率低。请在筛选区上方新增「平台快捷标签栏」：调用 `GET /api/influencers/platforms` 渲染全部平台为可点击标签，支持单选和多选组合（再次点击取消选中），选中标签与下方平台下拉双向同步；标签栏在窄屏下可横向滚动且当前选中项自动滚入可视区域。筛选变更后分页回到第一页。用 operator 同时选中「抖音」和「小红书」验证列表结果仅含这两个平台，清空标签后下拉框也应同步重置。

### F-03
合作列表查看单条记录的效果数据必须打开完整编辑 Modal，运营查阅成本高。请为合作表格增加行展开能力：行首增加展开箭头，点击后在当前行下方内联展示效果数据四宫格（浏览量/点赞/评论/分享）、备注摘要（超过 80 字省略）和最后更新时间；同一时刻仅允许展开一行，翻页或切换筛选后自动收起所有展开行。用 user 账号展开一条含效果数据的种子合作，确认四宫格数值与 Modal 内一致；用 operator 修改效果数据保存后，不刷新页面再次展开应看到更新后的值。

### F-04
数据看板「平台分布」饼图目前仅为静态展示。请为饼图扇区增加交互：hover 时 Tooltip 展示平台名称、网红数量和占比百分比；点击扇区后跳转网红列表并自动选中对应平台筛选（与平台快捷标签栏或下拉联动）。扇区颜色与平台保持固定映射，图例点击可隐藏/显示对应扇区。用 admin 登录看板，点击「B站」扇区后确认网红列表仅展示 B 站达人，返回看板后图表数据与 `/api/statistics/platform-distribution` 返回值一致。

### F-05
网红详情页底部合作记录表格展示全部历史合作，记录多时难以定位。请在合作记录区域上方增加状态筛选下拉（全部/待开始/进行中/已完成/已取消），筛选仅作用于详情页内表格不触发整页重载；表格底栏增加「在合作列表中查看全部」链接，点击后跳转合作列表并带上 `influencer_id` 筛选。用 operator 打开「李美妆」详情，筛选「已完成」后行数应减少，点击底栏链接后合作列表顶部筛选区应锁定该网红且结果集与详情筛选后一致。

### F-06
用户管理页无法快速了解某运营人员负责了哪些项目。请在后端合作列表接口补充 `created_by`（或等效）查询参数，按创建人 user_id 过滤；用户管理表格每行新增「合作数」徽章和「查看合作」链接，徽章数字为该用户创建的合作总数，点击链接跳转合作列表并自动按该用户筛选。用 admin 查看 operator 账号所在行，徽章数应与数据库中 `collaborations.user_id` 统计一致，跳转后列表仅展示该用户创建的合作。

### F-07
个人中心目前只有基本信息和改密码两个 Tab，运营无法概览自己的工作量。请新增第三个 Tab「我的合作」：展示当前登录用户创建的合作按状态分组的统计卡片（各状态数量）、本月创建数与累计预算/实际支出汇总，以及最近 5 条合作简表（项目名、状态、预算、日期）；简表项目名可点击跳转合作列表并以项目名关键词筛选。分别用 operator 和 user 登录，operator 应看到非空统计与简表，user 若未创建合作则展示空状态插画和引导文案。

### F-08
登录页密码框无法确认输入内容，注册时也无法感知密码强度。请为登录和注册模式的密码输入框增加「显示/隐藏」切换按钮（眼睛图标）；注册模式下在密码框下方增加实时强度指示条，按弱/中/强三档着色（依据长度、是否含数字、是否含字母组合等规则），强度为「弱」时注册按钮 disabled 并在下方提示具体原因。验证：输入纯数字短密码时指示为弱且无法提交，输入符合规则的密码后指示为中或强且可正常注册。

### F-09
侧边栏在笔记本小屏上占用过多横向空间。请为 Layout 侧边栏增加折叠/展开功能：顶栏或侧边栏底部增加折叠按钮，折叠后宽度缩至仅显示菜单图标（约 64px），主内容区自适应变宽；折叠状态持久化到 localStorage，刷新页面后保持；折叠态下鼠标悬停菜单图标显示菜单名称 tooltip，展开态 tooltip 不显示。用 admin 登录后折叠侧边栏，刷新页面确认仍为折叠态，点击各图标可正常跳转且页面标题正确更新。

### F-10
运营需批量处理失效网红时只能逐条编辑，效率低。请为网红列表增加多选能力：表头增加全选 checkbox，每行增加单选 checkbox，勾选后表格上方浮现批量操作栏，提供「设为活跃」「设为停用」两个按钮；点击后弹出确认框展示即将影响的条数，确认后批量调用更新接口，完成后 Toast 汇报成功与失败条数并刷新列表。user 角色不显示 checkbox 列。用 operator 勾选 3 条网红批量设为停用，确认列表状态标签即时更新且未勾选的行不受影响。

### F-11
合作实际支出超过预算时缺乏前端预警，容易造成超支遗漏。请在合作新建/编辑 Modal 中增加预算监控：当实际支出达到预算 90% 时表单内展示橙色提示条，达到或超过 100% 时展示红色提示条并默认阻止提交，除非运营勾选「确认超支继续保存」复选框；保存成功后合作列表该行预算/支出列旁对超支记录显示红色「超支」角标。用 operator 编辑一条预算 10000 的合作，将实际支出改为 10500，不勾选确认时应无法保存，勾选后保存成功且列表出现超支角标。

### F-12
分类数量增多后难以快速定位。请在分类管理页表格上方增加名称搜索框，支持按分类名称和描述模糊匹配（优先对接后端 keyword 参数，若暂无则前端实时过滤）；搜索结果中匹配的关键词高亮显示，无结果时展示空状态与「清除搜索」按钮。搜索不应影响新增和编辑 Modal 中的父级分类下拉数据源（下拉始终展示全部分类）。用 operator 输入「美妆」确认仅展示相关分类且关键词高亮，清除后恢复完整列表。

### F-13
数据看板「最近合作」表格缺少交互能力。请为该项目名列增加点击行为，点击后跳转合作列表并以该项目名作为关键词自动筛选；为「开始日期」和「预算」列头增加点击排序（升序/降序切换，显示方向箭头），排序在前端对当前展示的最近合作数据集生效。用 admin 登录看板，点击某条最近合作的项目名后合作列表应仅展示匹配项目，返回看板切换预算降序后金额最高的合作应排在最前。

### F-14
网红创建时平台与分类下拉在选项多时难以查找。请将创建/编辑 Modal 中的平台选择改为可输入过滤的 ComboBox（输入时实时过滤平台列表，支持键盘上下选择），选中平台后在输入框下方展示该平台账号格式提示文案（如抖音展示「@用户名 格式」）；分类下拉增加首字母/关键字快速定位能力，输入时跳转到首个匹配项。用 operator 创建网红时在平台框输入「小」应过滤出「小红书」，选中后看到格式提示，分类框输入「美」应定位到「美妆」。

### F-15
合作列表无法按投放金额区间检索项目。请在后端 `GET /api/collaborations` 增加 `min_budget` 和 `max_budget` 查询参数，前端筛选区增加「预算区间」输入（最低-最高，单位元，支持只填一侧），与现有状态、网红、内容类型筛选组合使用时变更后分页重置第一页，输入非数字或下限大于上限时给出字段级错误提示不发起请求。用 operator 设置预算 5000-20000 筛选，确认结果集中每条合作 budget 均落在区间内，与仅按状态筛选组合使用结果正确。

### F-16
三个列表页分页固定每页 10 条，数据量大时翻页频繁。请扩展分页组件支持每页条数切换（10/20/50），选择后写入 sessionStorage 并在网红列表、合作列表、用户列表三处分别记忆（互不影响），变更 page_size 时回到第一页并传给后端；当前页码超过新总页数时自动校正到最后一页。用 admin 在用户列表切换为每页 50 条确认单页展示更多数据，切换到合作列表应为独立的条数记忆，刷新同模块后条数选择保持。

### F-17
运营长时间查看数据看板时希望获得更大可视区域。请在顶栏右侧增加全屏切换按钮，使用 Fullscreen API 将 `document.documentElement` 切换至浏览器全屏/退出，按钮图标随状态在「进入全屏」和「退出全屏」间切换；进入全屏后侧边栏和顶栏布局不应错位或出现双滚动条，按 ESC 退出全屏后按钮状态同步更新。用任意账号登录后点击全屏按钮，确认图表区域放大且页面功能正常，退出后布局恢复如初。

### F-18
网红列表「合作次数」列目前仅为静态数字。请将该列数字改为可点击链接样式（合作次数大于 0 时），点击后跳转合作列表并自动带上 `influencer_id` 筛选；合作次数为 0 时展示灰色「暂无」文本且不可点击。链接样式需与项目名称链接有明显区分（如下划线或次要色）。用 operator 点击粉丝数较高的种子网红的合作次数，确认跳转后合作列表仅展示该网红关联项目且条数与数字一致。

### F-19
看板顶部四张概览卡片缺少趋势感知。请为每张卡片补充环比副指标：基于 `monthly-trends` 或新增轻量接口计算本月与上月的变化率，在卡片主数字下方展示「较上月 +12%」或「-5%」文案，正增长绿色、负增长红色、无法计算时灰色展示「—」；副指标加载时卡片展示骨架占位避免布局跳动。用 admin 查看看板，四张卡片均应显示副指标且颜色语义正确，网络慢时骨架与正式数据切换无错位。

### F-20
注册流程对邮箱重复和协议确认缺乏约束。请为注册表单增加邮箱失焦校验：调用后端接口或注册错误信息判断邮箱是否已占用，已占用时输入框下方红色提示并阻止提交；增加「我已阅读并同意服务协议」勾选框，未勾选时注册按钮保持 disabled；勾选框旁「服务协议」可点击打开 Modal 展示静态协议正文。验证：用已存在邮箱注册应看到占用提示，不勾选协议时按钮不可点，完整填写合法信息并勾选后可注册成功。

## 0-1构建（20）

### N-01
**达人分级：**为网红资源建立可配置的等级体系，运营可自定义等级名称、展示颜色与粉丝数区间，并为每位达人指定所属等级。管理页面以卡片形式维护各等级规则，支持排序调整；网红列表与编辑表单中展示等级徽章，列表可按等级快速筛选，方便商务在选型时一眼识别头部、腰部与潜力达人。

### N-02
**内容交付物：**围绕合作项目沉淀每一条已发布内容的完整记录，包括发布平台、内容链接、上线时间与审核结论，支持待审、已通过、已驳回三种审核流转。合作列表提供交付物入口，侧栏以时间线呈现该项目的全部内容产出，可逐条补充链接、变更审核意见或移除无效记录；列表层支持一键筛出「仍有待审内容」的合作，便于运营集中处理。

### N-03
**报价历史：**在网红单次报价发生变动时自动留存变更轨迹，记录调价前后金额、操作人与变更时间。详情页专设报价走势区域，以折线图勾勒价格起伏，配合明细表展示每次涨跌幅度与变更背景，帮助商务在议价、续约时快速调取历史报价依据；该模块仅对运营与管理员开放。

### N-04
**品牌方门户：**为品牌方客户提供独立的只读访问入口，界面精简、不含内部管理菜单。管理员可为品牌账号授权可见的营销战役范围，门户内展示战役预算消耗进度、关联合作进展与效果汇总；达人联系方式等敏感字段自动脱敏，品牌方仅能查看与自身投放相关的信息，无法触及平台完整后台能力。

### N-05
**运营 KPI 看板：**从人效视角统计各运营人员在指定月份内的项目产出，涵盖发起合作数、完结项目数、累计预算与支出、平均千次曝光成本等核心指标。页面顶部可切换统计月份，主体以柱状图横向对比团队成员表现，辅以明细表呈现具体数值，点击某位成员可下钻至其负责的合作列表，便于管理层做绩效复盘与资源调配。

### N-06
**触达 Pipeline：**将商务拓展阶段的达人跟进过程可视化为漏斗看板，划分待联系、沟通中、报价确认、已签约、已放弃五个阶段。每张达人卡片展示名称、平台与粉丝规模，支持拖拽跨列变更所处阶段，点击卡片可在侧栏补充跟进备注与指定负责人，让团队在一张看板上掌握全部在谈达人的实时进展。

### N-07
**财务台账：**为每笔合作配套发票与回款信息的全流程登记，记录发票编号、开票金额与日期，并跟踪未回款、部分回款、已回款三种资金状态。财务页面以表格集中展示全部台账，未回款记录以浅色底纹醒目标识，页头汇总应收与已收总额；支持按回款状态筛选，方便财务同事按月核对账款、催收逾期款项。

### N-08
**竞品情报：**建立竞品品牌投放动态的采集与检索能力，记录竞品名称、关联达人、推测合作金额、信息来源与发现日期。独立情报页支持录入与关键词检索，网红详情页同步展示该达人相关的竞品合作记录，辅助运营在选型时评估达人是否已被竞品锁定、判断谈判优先级与报价策略。

### N-09
**内容排期日历：**以月历视角统筹各合作项目的计划发布时间，每条排期关联具体合作、发布平台与内容类型，区分计划中、已发布、已逾期三种执行状态。日历格上以颜色标点标示排期密度，点击某日可展开当天待发布清单，逾期未上线的内容以红色醒目标记，帮助内容运营把控发布节奏、避免档期冲突与漏发。

### N-10
**消息模板库：**沉淀商务日常高频使用的沟通话术，按初次邀约、跟进催复、合同确认等场景分类管理。模板正文支持嵌入达人姓名、所属平台等动态字段，在模板管理页可自由增删改；网红详情页提供一键生成邀约功能，选中模板后即时预览替换变量后的完整文案，减少重复撰写、统一对外沟通口径（无需对接真实邮件发送）。

### N-11
**多平台账号绑定：**允许一位达人档案下挂载多个平台账号，各自记录平台名称、账号标识与粉丝规模，并指定其中一个为主展示账号。详情页以列表形式管理全部关联账号，可随时切换主账号；网红列表的平台列展示主账号信息，若另有绑定则附带数量角标，解决同一达人跨平台运营时信息分散、档案不完整的问题。

### N-12
**季度预算规划：**按年度与季度为各投放平台设定预算上限，并随合作创建与支出更新自动累计已用额度。预算管理页以进度条直观呈现各平台消耗比例，接近预警线时变色提示，超额时标红警示；运营在录入合作金额时，若将导致该平台季度预算超支，表单内即时弹出预警提示，帮助团队在花钱前感知余量、避免盲目投放。

### N-13
**合作评价：**对已完结的合作项目建立结构化评分机制，从内容质量、配合程度、投放效果三个维度各打一至五星，并附文字评语；每条合作仅可评价一次。合作列表为已完结项目提供评价入口，网红详情页汇总历史评分并以雷达图呈现各维度均分，为后续续约、拉黑或优先推荐提供可量化的履约参考。

### N-14
**经营快照：**支持将当前数据看板的核心经营指标（网红总量、合作规模、预算与互动汇总等）以自定义名称存档留存。快照列表可浏览历史存档记录，勾选任意两次快照进行并列对比，以表格形式展示各指标的变化差值与涨跌幅度，方便管理层在周报、月报或复盘会议中引用历史节点数据、追踪经营趋势。

### N-15
**地域分布：**为达人档案补充所在省份信息，打通区域化选型与统计分析能力。网红列表增加省份筛选条件，数据看板增设地域分布图，以横向柱状展示各省份达人数量对比，点击某一省份可直达对应地区的达人列表；帮助有区域投放需求的团队快速圈定目标省份的可用达人资源。

### N-16
**协作任务：**为合作项目的执行过程配备可拆分的待办清单，每条任务包含标题、负责人、截止日期与完成状态。独立任务页汇总展示全部未完成事项，合作列表提供快捷入口打开该项目的任务侧栏，支持勾选完成、追加新任务；顶栏角标实时提示已逾期任务数量，避免关键节点遗漏、提升多人协作时的跟进透明度。

### N-17
**敏感操作二次验证：**对删除用户、删除达人、删除合作三类高危操作增加密码复核环节，执行前须输入当前登录密码确认身份，校验不通过则阻断操作并给出明确提示。前端删除确认弹窗集成密码输入区域，后端同步校验，降低误删与越权删除风险，为关键数据提供最后一道安全防线。

### N-18
**达人推荐：**根据投放需求智能匹配最合适的达人候选，支持按平台、内容分类、粉丝区间、预算上限、最低互动率等条件组合筛选。系统按规则加权打分后返回推荐排序列表，每位候选人附带匹配理由标签（如报价合适、互动表现突出、历史合作丰富等）；推荐页以卡片形式呈现结果，可一键跳转达人详情进一步评估，缩短从提需求到出人选的决策路径。

### N-19
**新手引导：**为初次使用平台的用户提供分步式功能导览，首次登录后自动弹出引导遮罩，依次高亮侧边栏各功能菜单并配以简要说明，引导过程中可逐步前进或一键跳过。完成或跳过后不再重复打扰，个人中心保留「重新查看引导」入口，方便新入职同事或久未使用的成员随时重温操作流程。

### N-20
**系统设置：**将分散在代码中的平台行为参数集中为可配置项，由管理员在专属设置页统一维护，涵盖列表默认每页条数、是否开放自助注册、合作到期默认提醒天数、各平台对外显示名称等。修改保存后全局即时生效——分页默认值、登录页注册入口可见性、平台名称展示等均随配置联动更新，无需改代码即可灵活调整系统运行策略。
