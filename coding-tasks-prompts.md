## Feature迭代（20）

### F-01
网红列表页目前只有关键词、平台、分类和状态筛选，后端 `GET /api/influencers` 已支持 `min_followers` 和 `max_followers` 参数但前端未对接。请在网红列表筛选区增加粉丝数区间输入（支持只填下限或只填上限），筛选结果需与分页联动重置；同时在列表表头增加粉丝数列的升序/降序切换，排序状态切换后应重新请求列表。完成后用 operator 账号登录，分别用「10万-50万」「100万以上」等组合验证筛选与排序是否生效。

### F-02
数据看板页已请求 `category-distribution` 接口但页面未渲染分类分布数据。请在看板中新增一块「分类分布」可视化区域（饼图或柱状图均可），展示各分类下的网红数量占比；图表中的分类名称需可点击，点击后跳转到网红列表并自动带上对应 `category_id` 筛选条件。用 admin 登录后在看板确认图表有数据、点击分类后列表筛选正确。

### F-03
看板「Top 网红」区域目前固定按粉丝数排序。请增加排序维度切换控件，支持「粉丝数」「合作次数」「互动率」三种排序，分别对接 `GET /api/statistics/top-influencers?order_by=followers|collaborations|engagement`；切换时榜单应有过渡加载状态，榜单项需展示对应维度的数值与单位。登录后在看板依次切换三种排序，确认榜单顺序与数值符合预期。

### F-04
用户管理页目前展示全部用户但无搜索能力，后端已支持 `keyword`、`role_id`、`status` 查询参数。请在用户列表上方增加搜索栏：关键词（匹配用户名/昵称/邮箱）、角色下拉、状态下拉，搜索与重置按钮；筛选变更后分页回到第一页。用 admin 账号搜索「operator」、筛选「运营」角色，确认表格结果正确且分页总数同步变化。

### F-05
合作列表页缺少时间维度筛选，后端 `GET /api/collaborations` 支持 `start_date_from` 和 `start_date_to`。请在筛选区增加合作开始日期的起止日期选择器，并提供「本周」「本月」「本季度」三个快捷预设按钮，点击预设后自动填入日期范围并触发查询。用 operator 创建或利用种子数据中不同日期的合作记录，验证快捷预设与手动选日期筛选结果一致。

### F-06
个人中心基本信息 Tab 中角色展示写死为「普通用户」，未读取当前登录用户的真实角色。请修复角色展示逻辑，从认证上下文读取 `role` 并映射为中文（管理员/运营/普通用户）；同时在基本信息区域下方增加一块「当前权限说明」面板，按角色列出可访问的菜单与可执行的操作（如增删改网红、管理用户等）。分别用 admin、operator、user 三个账号进入个人中心，确认角色名称与权限说明内容各自正确。

### F-07
后端已有 `PUT /api/profile/avatar` 接口可更新头像 URL，但个人中心无编辑入口。请在个人中心基本信息 Tab 增加头像编辑功能：支持输入图片 URL、实时预览、保存与取消；保存成功后顶栏与侧边栏的用户头像同步更新；若 URL 无效或加载失败则回退到默认头像占位图。用任意账号修改头像 URL 并刷新页面，确认头像持久化且各位置展示一致。

### F-08
分类模型支持 `parent_id` 树形结构，但分类管理页以扁平卡片展示且无父级选择。请将分类列表改为树形展示（子分类缩进或折叠），新增/编辑分类 Modal 中增加「父级分类」下拉（顶级分类可选「无」）；删除时若存在子分类应给出明确提示。用 operator 创建「美妆 > 护肤 > 面膜」三级结构，确认列表层级正确、编辑父级后结构更新。

### F-09
网红详情页目前只能查看不能编辑，运营人员需返回列表才能修改。请在网红详情页为 operator 和 admin 增加「编辑档案」按钮，点击后弹出与列表页一致的编辑 Modal（预填当前数据）；保存成功后详情页各字段即时刷新，无需整页重载。用 operator 从详情页修改报价、标签和备注，确认保存后详情与列表数据一致。

### F-10
合作状态变更目前必须打开完整编辑 Modal。请在合作列表每行增加状态快捷操作：根据当前状态显示可流转的下一状态按钮（如 pending → in_progress），点击后弹出简短确认框，确认后调用 `PUT /api/collaborations/{id}/status`；状态更新成功后该行状态标签颜色即时变化，并在行内展示最近一次状态变更时间（若后端无该字段需补充）。用 operator 将一条 pending 合作推进到 in_progress 再标记 completed，全程在列表完成操作并验证。

### F-11
网红列表不支持按报价和互动率排序。请在后端列表接口增加 `sort_by`（followers/cost_per_post/engagement_rate）和 `sort_order`（asc/desc）参数，前端表头「粉丝数」「单次报价」「互动率」列支持点击切换排序，当前排序列显示方向指示箭头；排序与筛选、分页可同时生效。验证：筛选「抖音」平台后按互动率降序，确认结果集内排序正确。

### F-12
合作编辑 Modal 未展示创建人信息，运营协作时无法知晓项目发起人。请在合作 Modal 的查看/编辑模式下展示创建人昵称与创建时间；若当前用户为创建人则显示「我创建的」标识。另在合作列表增加「创建人」列（显示昵称，过长省略）。用 admin 和 operator 分别创建合作，切换账号查看列表与 Modal，确认创建人信息准确。

### F-13
看板「月度趋势」图表固定请求 6 个月数据。请在图表标题旁增加时间跨度选择（3个月/6个月/12个月），切换时重新请求 `monthly-trends?months=` 并刷新折线图；鼠标悬停数据点时在 Tooltip 中展示当月合作数、预算、实际支出三项明细。登录看板后切换 3 与 12 个月，确认曲线数据点数量与 Tooltip 内容正确。

### F-14
登录页每次需重新输入用户名。请增加「记住用户名」勾选项，勾选并登录成功后将用户名存入 localStorage，下次打开登录页自动填充；同时增加「最近登录账号」下拉（最多保存 3 个），可快速切换填充。验证：用 admin 勾选记住并登录，关闭标签页重新打开确认用户名已填充；再用 operator 登录一次，下拉中应出现两个账号可选。

### F-15
普通用户（user 角色）在合作列表仍能看到编辑/删除按钮（或 Modal 内可编辑字段），体验不符合只读定位。请梳理合作模块前端权限：user 角色隐藏所有写操作按钮，点击合作标题或「查看」仅打开只读 Modal（所有表单项 disabled）；只读 Modal 内为效果数据（浏览/点赞/评论/分享）增加迷你柱状对比图。用 user 账号登录验证无法触发任何写操作，且效果数据图表正常展示。

### F-16
网红创建/编辑 Modal 缺少头像 URL 字段，列表头像列多为占位图。请在 Modal 表单中增加头像 URL 输入与预览；列表表格头像列展示真实缩略图（40×40 圆角），图片加载失败显示平台首字母占位；滚动长列表时头像不应明显卡顿。创建一条带有效头像 URL 的网红记录，确认列表与详情页均展示真实头像。

### F-17
分类有 `sort_order` 字段但前端无法调整顺序。请在分类管理页支持拖拽排序：按住分类卡片拖拽可调整同级顺序，松手后批量调用更新接口保存 `sort_order`；保存中显示 loading，失败时回滚到拖拽前顺序并 Toast 提示。用 operator 调整三个同级分类顺序，刷新页面后顺序应保持。

### F-18
合作列表缺少投放效果量化指标。请在前端为每条合作计算并展示「千次曝光成本 CPM」（实际支出/浏览量×1000，浏览量为 0 时显示「—」）和「互动率」（(点赞+评论+分享)/浏览量，浏览量为 0 时显示「—」）；CPM 低于 50 显示绿色，50-200 显示黄色，高于 200 显示红色。用种子数据中不同效果数据的合作行验证计算与着色逻辑。

### F-19
看板顶部四张概览统计卡片（网红总数、合作总数、粉丝覆盖、预算总额）目前仅为静态展示。请为每张卡片增加点击跳转：网红总数 → 网红列表，合作总数 → 合作列表，粉丝覆盖 → 网红列表并自动按粉丝数降序，预算总额 → 合作列表并自动按预算金额降序；卡片 hover 时应有可点击视觉反馈（手型光标、轻微阴影）。用 admin 登录看板，依次点击四张卡片，确认跳转目标页面与自动携带的排序或筛选参数正确。

### F-20
注册流程成功后仅提示注册成功，用户需手动再点登录。请将注册成功后的流程改为：注册接口返回成功后自动调用登录接口获取 token，写入认证上下文并跳转看板；首次注册成功的用户在看板顶部展示一次性欢迎 Toast（3 秒自动消失，含用户名）。用新邮箱注册一个新账号，确认无需二次点击登录即可进入系统并看到欢迎提示。

## 0-1构建（20）

### N-01
平台缺少独立的合作详情页，运营无法通过 URL 分享或收藏某条合作。请新建 `/collaborations/:id` 合作详情页：顶部展示项目名称、状态徽章、内容类型；主体分左右两栏——左侧为合作基本信息（时间、预算、支出、效果数据、备注），右侧为关联网红档案卡片（头像、平台、粉丝数、快捷跳转）；底部展示效果数据趋势占位区（用静态迷你图即可，数据来自当前合作）。从合作列表、看板最近合作均可跳转至此页，浏览器刷新后数据不丢失。

### N-02
平台目前无法追溯「谁在什么时候改了什么」。请新建操作审计日志模块：后端新增 `audit_logs` 表（操作者、模块、动作、目标 ID、变更摘要、IP、时间），在网红/合作/分类的创建、更新、删除时自动写入日志；前端新建「操作日志」页面（仅 admin 可访问），支持按操作者、模块、时间范围筛选与分页，列表展示可读的中文摘要（如「张三 将合作『春季推广』状态改为已完成」）。用 operator 修改一条合作后，admin 在日志页能查到对应记录。

### N-03
头像与附件目前只能填 URL，缺少本地上传能力。请新建文件上传服务：后端提供 `POST /api/uploads/image`（限制 jpg/png、最大 2MB），文件存储到后端 `uploads/` 目录并返回可访问 URL；前端在个人中心头像编辑和网红 Modal 头像字段旁各增加「本地上传」按钮，上传中显示进度，成功后自动填入 URL 并预览。上传一张图片后复制返回 URL 在新标签页打开，确认图片可访问。

### N-04
运营需要把网红库导出给品牌方评审。请新建数据导出功能：后端 `GET /api/influencers/export?format=csv` 按当前筛选条件导出（支持与列表相同的 keyword/platform/category_id/status/粉丝区间参数），CSV 含表头中文别名；前端网红列表页增加「导出 CSV」按钮，点击后下载文件，导出过程中按钮置灰防重复点击。先筛选「小红书」平台再导出，打开 CSV 确认仅含小红书网红且字段完整。

### N-05
手工逐条录入网红效率太低。请新建批量导入模块：后端提供 CSV 模板下载接口和 `POST /api/influencers/import` 上传接口，解析后逐行校验（必填项、平台枚举、分类名匹配），返回成功/失败条数及失败行原因；前端网红列表页增加「批量导入」入口，流程为下载模板 → 上传文件 → 展示导入结果报告（成功绿色、失败红色可展开原因）。准备一份含 3 条合法和 2 条非法数据的 CSV 验证导入报告准确。

### N-06
运营经常需要标记重点网红但系统无收藏能力。请新建收藏模块：后端新增 `favorites` 表（user_id + influencer_id 唯一约束）及收藏/取消/列表 API；前端在网红列表和详情页增加星标收藏按钮（已收藏实心、未收藏空心），新建「我的收藏」页面（路由 `/favorites`）展示当前用户收藏的网红列表，支持取消收藏。用 operator 收藏 2 个网红，在「我的收藏」页确认展示，换 user 账号登录确认看不到他人的收藏。

### N-07
信息分散在网红、合作、用户多个模块，缺少统一入口。请新建全局搜索：后端 `GET /api/search?q=` 同时检索网红（名称/账号）、合作（项目名/品牌名）、分类（名称），返回分组结果与跳转路径；前端顶栏增加搜索框，输入关键词后下拉展示分组搜索结果（每类最多 5 条），点击条目跳转到对应详情或列表筛选页，按 ESC 或点击空白关闭。搜索「李佳」应能同时出现网红和含该网红的合作（若种子数据有）。

### N-08
合作状态变更后相关人员无法及时知晓。请新建通知中心：后端新增 `notifications` 表，在合作状态变更、网红被加入黑名单时向相关用户写入通知；前端顶栏增加铃铛图标与未读数角标，点击展开最近 10 条通知下拉，新建「通知中心」页 `/notifications` 支持全部已读、单条已读和分页。用 admin 将 operator 创建的合作状态改掉，operator 登录后铃铛应出现未读通知。

### N-09
合作项目涉及合同但系统无处存放。请新建合同管理模块：后端新增 `contracts` 表（关联 collaboration_id、合同编号、签约日期、合同金额、附件 URL、备注），提供 CRUD API；前端新建独立的「合同管理」页 `/contracts`，以表格展示全部合同，支持按合作项目名称搜索，并可从合作列表行内「管理合同」按钮跳转过来且自动过滤到该合作的合同记录。为一条合作添加合同编号「HT-2026-001」并保存，在合同管理页能检索到该记录。

### N-10
运营团队在合作项目上需要内部讨论但无留言功能。请新建协作备注模块：后端新增 `collaboration_comments` 表（collaboration_id、user_id、content、created_at），支持增删（仅作者或 admin 可删）；前端在合作列表每行增加「讨论」按钮，点击后打开侧栏面板展示该合作的内部讨论时间线（倒序），底部可发表新评论，支持 `@昵称` 纯文本高亮。两人账号对同一合作各发一条评论，确认侧栏时间线顺序与删除权限正确。

### N-11
合作排期分散在列表中不直观。请新建活动日历页 `/calendar`：读取所有合作的起止日期，在月历视图上以条形事件展示（不同状态不同颜色），点击某日事件弹出当天合作列表侧栏，点击单条可跳转合作详情；支持上月/下月切换。确保跨月合作在起止两天及中间日期均有展示。用种子数据打开日历页，确认有多条合作事件且颜色与状态对应。

### N-12
品牌方关注投放 ROI 但看板只有宏观汇总。请新建 ROI 分析页 `/analytics/roi`：筛选时间范围后，展示合作维度的 CPM/CPE（单次互动成本）排行榜、预算 vs 实际支出对比柱状图、按平台分组的平均 ROI 雷达图或表格；数据由前端聚合已有合作接口或新增轻量统计 API。选择「近 6 个月」后页面应有多张图表且有真实数据填充。

### N-13
选型网红时需要并排对比。请新建网红对比工具页 `/influencers/compare`：页面提供网红多选器（最多选 4 个），选中后以对比表格展示粉丝数、平台、报价、互动率、历史合作次数、平均合作效果；增加「雷达图」对比维度（粉丝/互动率/合作次数/报价倒数归一化）。从网红列表勾选 3 人点击「加入对比」跳转对比页，确认表格与雷达图数据正确。

### N-14
网红标签目前是逗号分隔字符串，无法统一管理。请新建标签管理模块：后端新增 `tags` 表和 `influencer_tags` 多对多关联，提供标签 CRUD 和为网红打标 API；前端新建「标签管理」页 `/tags`（operator 可访问），可创建标签（名称+颜色）；网红编辑 Modal 改为标签多选器（支持搜索已有标签、快速新建）。创建「美妆」「探店」标签并为网红打上，列表页能以标签筛选（新筛选控件）。

### N-15
大额合作创建后缺少审批环节。请新建审批工作流：合作新增状态 `pending_approval`，operator 创建合作后默认进入该状态且不可直接改为 in_progress；admin 在合作列表看到「待审批」筛选和「通过/驳回」按钮，通过后变为 pending，驳回后变为 cancelled 并记录驳回原因；operator 在被驳回的合作详情可看到原因。完整走一遍创建→审批通过→开始执行的前端流程。

### N-16
合作仅有 `project_name` 字符串，缺少品牌/战役维度管理。请新建品牌战役模块：后端新增 `campaigns` 表（名称、品牌方、预算上限、起止日期、状态），合作表增加 `campaign_id` 外键；前端新建「战役管理」页 `/campaigns`，可 CRUD 战役，合作 Modal 增加战役下拉选择；战役详情页展示关联合作列表与预算使用率进度条。创建一个「618大促」战役并关联 2 条合作，详情页进度条应反映已用预算比例。

### N-17
